## Feature Prioritisation Framework

### Must Have (P0)

- Core functionality works
- Security and privacy compliant
- Mobile experience good
- Shop data accurate

### Should Have (P1)

- Shop owner tools
- Enhanced search/filter
- Review features
- Notifications

### Nice to Have (P2)

- Social features
- Advanced discovery
- Integrations

### Future (P3)

- Expansion features
- Premium advanced features
- API platform

---

## Decision Criteria

Before adding features, ask:

1. **User value**: Does this help visitors find/save/review shops?
2. **Business value**: Does this help shop owners or generate revenue?
3. **Effort**: Can we build it in 1-2 weeks?
4. **Maintenance**: Can we maintain it long-term?
5. **Simplicity**: Does it keep the product simple?

## Risks and Mitigations

| Risk | Mitigation |
|------|------------|
| Low shop owner adoption | Personal outreach, demonstrate value |
| Low user engagement | Review prompts, notifications, content |
| Competition | Move fast, build relationships, local focus |
| Technical issues | Good architecture, monitoring, quick fixes |
| Revenue challenges | Start premium early, keep costs low |
